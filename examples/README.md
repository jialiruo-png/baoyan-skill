# Examples README

本文件夹用于存放商科 / 经管 / 教育 / 公共管理 / 新闻传播等保研、夏令营、预推免个人陈述案例。

这些案例的主要作用不是提供可直接套用的模板，而是作为**结构、语气、专业迁移和篇幅控制**的参考样本，帮助生成更贴合中国高校推免场景的中文个人陈述。

---

## 一、使用目的

本文件夹中的案例主要用于以下几类参考：

1. **结构参考**  
   观察不同个人陈述如何组织：
   - 开头方式
   - 小标题设计
   - 学术背景展开
   - 科研经历铺陈
   - 实践经历取舍
   - 未来规划写法

2. **语气参考**  
   观察不同案例如何保持：
   - 正式
   - 克制
   - 学术化
   - 真诚但不煽情

3. **专业适配参考**  
   观察申请不同专业时，如何调整：
   - 关键词
   - 研究兴趣表述
   - 专业匹配段
   - 未来规划方向

4. **字数规格参考**  
   观察 1000 / 1500 / 2000 / 2500 / 3000 字左右的文书，如何在篇幅变化下调整信息密度。

## 二、使用原则

### 1. 优先借鉴“写法”，不要照抄“内容”
这些案例主要用于借鉴：
- 结构安排
- 表达逻辑
- 语气风格
- 专业迁移方式
- 小标题形式
- 研究与实践的组织方式

不要直接照抄：
- 具体事实
- 项目名称
- 研究结论
- 学校名称
- 奖项经历
- 原句原段

---

### 2. 案例不是硬模板
案例之间可能存在：
- 专业差异
- 风格差异
- 长度差异
- 学术导向差异
- 申请层次差异

如果不同案例之间存在不一致之处，应以：
1. `SKILL.md`
2. `references/structure-guide.md`
3. `references/tone-guide.md`
4. `references/writing-pitfalls.md`

中的规则为准。

---

### 3. 案例只作为“风格校准”和“结构对照”
案例的作用是帮助判断：
- 这类申请材料通常怎么写
- 某种专业适合突出什么
- 某种背景适合怎样搭主线
- 哪种表述更像中国高校推免个人陈述

案例不应取代：
- 用户真实材料
- 目标项目的官方信息
- 当前申请任务的专业匹配分析

---

### 4. 目标专业较特殊时，不要只靠案例
如果用户申请的是较新、较细、较交叉的专业，或目标院校项目具有明显特色，应优先：
- 检索官方项目介绍
- 查看学院 / 学部官网
- 识别培养目标与研究重点

然后再参考案例调整写法。

---

## 三、文件组织原则

本文件夹中的案例采用：

**一篇案例 = 一个独立 `.md` 文件**

不建议把所有个人陈述合并到一个大文件中，因为那样会导致：

- 不利于维护
- 不利于定位
- 不利于按专业筛选
- 容易让模型混淆不同案例的结构与风格

每篇案例文件开头可包含简要元信息，例如：
- 标题
- 申请专业
- 估计字数
- 是否带小标题
- 主要可借鉴点

---

## 四、案例命名建议

文件名尽量简洁、可检索、可扩展。

推荐命名格式：

`ps-[方向]-[序号].md`

例如：
- `ps-edu-econ-01.md`
- `ps-agri-econ-02.md`
- `ps-public-management-07.md`
- `ps-international-communication-08.md`

如果后续案例增多，也可以按方向继续细分：
- `ps-carbon-econ-01.md`
- `ps-digital-econ-01.md`
- `ps-finance-01.md`

---

## 五、如何使用这些案例

在生成个人陈述时，优先做以下事情：

1. 先读取用户真实材料  
2. 判断目标学校、学院、专业与字数要求  
3. 搭建当前任务的叙事主线  
4. 仅在需要时参考本文件夹中的相似案例：
   - 参考结构
   - 参考小标题样式
   - 参考专业转写逻辑
   - 参考篇幅分配方式

正确顺序应当是：

**用户材料优先 → 任务目标优先 → 案例辅助参考**

而不是反过来。

## 七、后续维护建议

随着案例数量增加，建议定期做以下维护：

1. 补充案例索引  
2. 删除重复度过高的案例  
3. 对案例按专业方向重新归类  
4. 保留“代表性强”的案例，减少“信息价值低”的案例  
5. 对过长案例增加简要说明，标出可借鉴重点  

如果未来案例很多，也可以进一步拆分为子目录，例如：

- `examples/economics/`
- `examples/management/`
- `examples/education/`
- `examples/communication/`

## 八、一句话原则

**案例是参照物，不是模板库。**  
应借其结构与逻辑，不应复制其事实与语言。
